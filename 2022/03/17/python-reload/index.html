Python模块热更新

发表于 2022-03-17 分类于 Python
本文字数： 5.8k 阅读时长 ≈ 11 分钟

背景

  • 项目中需要插件化机制进行解耦，统一抽象交互接口，只要符合统一接口即可做到插件的替换 ，动态更新

分析

  • Python 作为动态语言提供了很灵活的importlib模块导入， 借此实现动态导入某模块

  • 根据“你一定不是第一个遇到此问题的人”法则，Github先搜为敬 ，找到一些比较活跃的库

    • https://github.com/julvo/reloading : 函数级热重载
    • https://github.com/breuleux/jurigged 热补丁到进程
    • https://github.com/samuelcolvin/watchgod 使用文件轮询监控文件变化，因为足够快且简单

  • 重新梳理需求

    • 初始化 某一目录下 所有模块
    • 随着指定目录里 python文件的 添加 ，删除，更新 而重新加载模块
    • 每一个模块当作一个插件，约定规范 以实现在主函数里统一调用

  • 现成的方案不太适合，这里用import 机制 配合文件变化 封装一下

实现

  • 从指定路径加载module

  • 使用 watchgod 轮询 监听文件变化，重载模块

  • 定义模块必须包含 _plugin_name, _plugin_version, run 方法进行 检测

  • 完整代码
